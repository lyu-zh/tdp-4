# 代码问题诊断和修复

## 问题描述

用户发现：可以访问的总共只有109个点（或3个点），但是每个中心点周围都可以找到20个可以访问的临近点，这很不合理。

## 问题分析

### 1. 当前情况

根据验证结果（日期：2021-04-10）：
- **可访问的点数**: 3/3991
- **所有区域的基础点集大小**: 4（3个可访问点 + 1个中心点）
- **所有区域都包含相同的3个可访问点**: 点[0], 点[1], 点[2]

### 2. 问题根源

**主要问题**：`readXiVectorFromDemandMatrix` 方法中的坐标匹配容差太小

**代码位置**：第 117 行
```java
double tolerance = 1e-14;  // 容差太小，可能导致匹配失败
```

**问题说明**：
- 坐标匹配使用 `Math.abs(p.longitude - lon) < tolerance` 来判断
- 如果 CSV 文件中的坐标精度与 `unique_coordinates_list.csv` 中的坐标精度不同，匹配可能失败
- 容差 `1e-14` 太小，对于浮点数比较来说过于严格
- 如果匹配失败，`xi[i]` 保持为 0（默认值），导致很多点被错误地标记为不可访问

### 3. 修复方案

#### 修复 1: 增加坐标匹配容差

**修改前**：
```java
double tolerance = 1e-14;
```

**修改后**：
```java
double tolerance = 1e-10;  // 增加容差，提高匹配成功率
```

#### 修复 2: 添加匹配统计信息

添加调试信息，帮助诊断问题：
```java
System.out.println("xi 向量读取统计:");
System.out.println("  - 成功匹配的点数: " + matchedCount);
System.out.println("  - 未匹配的点数: " + unmatchedCount);
System.out.println("  - 可访问的点数 (ξ_i = 1): " + accessibleCount + "/" + n);
```

#### 修复 3: 输出未匹配的点的警告

对于 `value = 1` 但未找到匹配的点，输出警告：
```java
if (!found && value == 1) {
    unmatchedCount++;
    if (unmatchedCount <= 10) {  // 只输出前10个，避免输出过多
        System.out.println("警告: 未找到匹配的点 (经度:" + lon + ", 纬度:" + lat + ", 值:" + value + ")");
    }
}
```

### 3. 添加未匹配点的警告

**文件**: `src/TSPDualExtractorValidator.java`
**位置**: 第 135-145 行

**新增代码**：
```java
boolean found = false;
for (int i = 0; i < n; i++) {
    Point p = allPoints.get(i);
    if (Math.abs(p.longitude - lon) < tolerance && 
        Math.abs(p.latitude - lat) < tolerance) {
        xi[i] = value;
        matchedCount++;
        found = true;
        break;
    }
}

// 如果未找到匹配的点，记录警告（仅对 value = 1 的点）
if (!found && value == 1) {
    unmatchedCount++;
    if (unmatchedCount <= 10) {  // 只输出前10个未匹配的点，避免输出过多
        System.out.println("警告: 未找到匹配的点 (经度:" + lon + ", 纬度:" + lat + ", 值:" + value + ")");
    }
}
```

3. **数据完整性**：
   - `demand_matrix.csv` 中是否包含所有点的数据
   - 目标日期列是否存在且包含正确的数据
