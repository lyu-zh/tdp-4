# 基础点集记录功能说明

## 功能描述

在 `TSPDualExtractor.java` 中添加了基础点集记录功能，类似于 `TSPDualExtractorValidator.java` 中的记录方法，将每个区域的基础点集信息记录到 txt 文件中。

## 实现内容

### 1. 新增方法

#### `formatBasePointSetInfo()` 方法（第 727-752 行）

**功能**：格式化基础点集信息，生成与 `TSPDualExtractorValidator` 类似的格式

**参数**：
- `j`: 区域索引
- `basePointSet`: 基础点集
- `centerIdx`: 中心点索引
- `allPoints`: 所有点的列表
- `xi`: xi 向量

**输出格式**：
```
区域 j 的基础点集:
  点集大小: n
  点集详情:
    ✓ 点[i] (经度:xxx.xxxxxx, 纬度:yy.yyyyyy): ξ[i]=x [中心点/depot]
    ...
```

#### `saveBasePointSetInfoToFile()` 方法（第 762-810 行）

**功能**：将所有区域的基础点集信息保存到 txt 文件

**特点**：
- 按区域索引排序（因为并行处理，顺序可能不一致）
- 包含文件头和参数信息
- 格式与 `TSPDualExtractorValidator` 的输出格式一致

#### `extractRegionIndex()` 方法（第 812-825 行）

**功能**：从格式化的字符串中提取区域索引，用于排序

#### `Pair` 内部类（第 826-833 行）

**功能**：简单的键值对类，用于排序基础点集信息

### 2. 主程序修改

#### 在主循环中记录基础点集（第 620-624 行）

**修改内容**：
```java
// 【新增】记录基础点集信息
String basePointSetInfo = formatBasePointSetInfo(j, basePointSet, centerIdx, allPointsFinal, xiFinal);
synchronized (allBasePointSetInfo) {
    allBasePointSetInfo.add(basePointSetInfo);
}
```

#### 保存基础点集信息（第 681-683 行）

**修改内容**：
```java
// 【新增】保存基础点集信息到 txt 文件
String basePointSetInfoFile = "output/base_point_sets_p" + k + ".txt";
saveBasePointSetInfoToFile(allBasePointSetInfo, basePointSetInfoFile, n, k, xi);
System.out.println("基础点集信息已保存到: " + basePointSetInfoFile);
```

### 3. 数据结构

**新增变量**（第 604 行）：
```java
List<String> allBasePointSetInfo = Collections.synchronizedList(new ArrayList<>());
```

用于收集所有区域的基础点集信息（线程安全，因为并行处理）

`output/base_point_sets_p{k}.txt`，其中 `{k}` 是区域数

## 特点

1. **格式一致**：与 `TSPDualExtractorValidator` 的输出格式保持一致
2. **线程安全**：使用 `Collections.synchronizedList` 确保并行处理时的线程安全
3. **自动排序**：按区域索引排序，确保输出顺序一致
4. **详细信息**：包含每个点的索引、坐标、ξ_i 值，以及中心点标记

## 使用说明

运行 `TSPDualExtractor.java` 后，会自动生成基础点集信息文件：
- 文件名：`output/base_point_sets_p{k}.txt`
- 包含所有区域的基础点集详细信息
- 可用于验证和调试

## 注意事项

1. **并行处理**：由于使用并行流处理，基础点集信息的收集顺序可能不一致，但会在保存时按区域索引排序
2. **文件大小**：如果区域数很多，输出文件可能较大
3. **性能影响**：记录基础点集信息会增加少量内存和 I/O 开销，但影响很小
